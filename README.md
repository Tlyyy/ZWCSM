## 跨设备同步（无服务器）

有两种方式：

### 方式1：Supabase 房间同步（推荐）
1. 在 Supabase 的 SQL Editor 运行 `supabase-schema.sql`。
2. 打开网页的「同步」页，填写同一个房间号，例如 `zwcsm`。
3. 第一台设备点 **上传到云端**。
4. 其他设备点 **从云端拉取**。
5. 需要自动同步时，点 **开启自动同步**。

### 方式2：无 Token 同步码
1. 在「同步」页展开 **同步码备用方案**，点击 **复制同步码**。
2. 另一台设备打开页面后，点 **从同步码恢复**，粘贴并确认即可恢复全部评分、类别和本周菜单。

### 方式2：GitHub 云同步（可选）
1. 打开网页「同步」标签。
2. 填写仓库：`owner/repo`（或粘贴完整仓库链接）。
3. 分支一般填 `main`，文件路径默认 `data/zwcsm-state.json`。
4. 填入个人 GitHub Token（至少有 `repo` 权限）。
5. 点击「保存配置」，再点「上传到云端」。
6. 另一台设备按同样配置并输入同一个 Token，点「从云端拉取」即可拿到同一份数据。

说明：
- 无 token 方式是“复制同步码/恢复同步码”。
- GitHub 方式是直接文件同步，不需要手工改 JSON。
